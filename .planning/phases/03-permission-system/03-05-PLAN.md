    Use imports: Request, Response, RequestHandler, ResponseFactory, RateLimitRepositoryInterface.

    Constructor: inject `RateLimitRepositoryInterface $rateLimitRepo` and `ResponseFactory $responseFactory` as private readonly promoted properties. Also accept optional $maxAttempts (default 5) and $windowSeconds (default 60) as constructor parameters with defaults.

    `process(Request $request, RequestHandler $handler): Response`:
    1. Get client IP: $clientIp = $request->getServerParam('REMOTE_ADDR', '0.0.0.0')
    2. Check rate limit: if $this->rateLimitRepo->isRateLimited($clientIp, $this->maxAttempts, $this->windowSeconds):
       - Create 429 response: $response = $this->responseFactory->createResponse(429)
       - Body: json_encode(['statusCode' => 429, 'error' => ['type' => 'RATE_LIMITED', 'description' => 'Too many login attempts. Please try again later.']], JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES)
       - Return $response->withHeader('Content-Type', 'application/json')
    3. Call $response = $handler->handle($request)
    4. Record attempt AFTER handler runs (counts both success and failure): $this->rateLimitRepo->recordAttempt($clientIp)
    5. Return $response

    Important: Record attempt AFTER the handler, not before. This ensures all attempts (successful logins too) count toward the limit.
  </action>

Do NOT use X-Forwarded-For or X-Real-IP (spoofable). Behind a reverse proxy, configure the proxy to set REMOTE_ADDR correctly. |
