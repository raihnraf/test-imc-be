  <action>
    Create `src/Application/Middleware/PermissionMiddleware.php` with namespace `Imc\Application\Middleware`, `declare(strict_types=1)`, implementing `Psr\Http\Server\MiddlewareInterface`.

    Use imports: Request, Response, RequestHandler, ResponseFactory, RouteContext (from Slim\Routing), PermissionRepositoryInterface.

    Constructor: inject `PermissionRepositoryInterface $permissionRepo` and `ResponseFactory $responseFactory` as private readonly promoted properties.

    `process(Request $request, RequestHandler $handler): Response`:
    1. Extract user_id and level_id from request attributes (set by JwtMiddleware):
       - $userId = (int) $request->getAttribute('user_id')
       - $levelId = (int) $request->getAttribute('level_id')
    2. Get the route: $route = $request->getAttribute(RouteContext::ROUTE)
    3. Get the page argument: $pagePath = $route?->getArgument('page')
    4. If $pagePath is null → return $handler->handle($request) (no permission check needed for this route)
    5. Call $this->permissionRepo->hasAccess($userId, $levelId, $pagePath)
    6. If returns false → create 403 response:
       - $response = $this->responseFactory->createResponse(403)
       - Body: json_encode(['statusCode' => 403, 'error' => ['type' => 'FORBIDDEN', 'description' => 'You do not have permission to access this resource']], JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES)
       - Return $response->withHeader('Content-Type', 'application/json')
    7. If returns true → return $handler->handle($request)

    Key behavior: Every request triggers a fresh DB query (no caching). This satisfies PERM-07 — permission changes take effect on the next request.
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JwtMiddleware → PermissionMiddleware | user_id and level_id attributes from JWT (trusted after validation) |
| PermissionMiddleware → PermissionRepository | pagePath from route argument (controlled by developer, not user input) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Elevation of Privilege | PermissionMiddleware | mitigate | user_id and level_id extracted from JWT attributes (set by JwtMiddleware, not from client input); hasAccess query uses these server-side values |
| T-03-10 | Tampering | Route page argument | mitigate | page argument set by developer in routes.php via ->setArgument(), not from client request; attacker cannot change which page is checked |
| T-03-11 | Denial of Service | PermissionMiddleware DB query | mitigate | Single SQL query per request with indexed columns (page_id, level_id, user_id); query is fast with proper indexes on level_permissions and user_permissions |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed |
</threat_model>
