  <action>
    Create `src/Application/Actions/Permission/PermissionMatrixAction.php` with namespace `Imc\Application\Actions\Permission`, `declare(strict_types=1)`, extending `BaseAction`.

    Constructor: inject `PermissionRepositoryInterface $permissionRepo`, `LevelRepositoryInterface $levelRepository`, and `UserRepositoryInterface $userRepository` as private readonly promoted properties.

    `__invoke(Request $request, Response $response, array $args): Response` — only handles GET:
    1. Parse query params: $levelId = $request->getQueryParam('level_id'), $userId = $request->getQueryParam('user_id')
    2. If level_id is provided (and user_id is not):
       a. Validate level exists → notFoundResponse('Level not found') if null
       b. Call $this->permissionRepo->getLevelMatrix((int) $levelId)
       c. Return jsonResponse with ['data' => $matrix, 'type' => 'level', 'level_id' => $levelId], status 200
    3. If user_id is provided (and level_id is not):
       a. Validate user exists → notFoundResponse('User not found') if null
       b. Get user's level_id from entity
       c. Call $this->permissionRepo->getUserMatrix((int) $userId, (int) $levelId)
       d. Return jsonResponse with ['data' => $matrix, 'type' => 'user', 'user_id' => $userId], status 200
    4. If neither is provided:
       a. Call $this->permissionRepo->getAllPages()
       b. Return jsonResponse with ['data' => array_map(fn($p) => [...$p, 'has_access' => null], $pages), 'type' => 'all'], status 200
    5. If both are provided: return validationErrorResponse with ['query' => ['Provide either level_id or user_id, not both']]
  </action>
