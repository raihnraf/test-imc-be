    `__invoke(Request $request, Response $response): Response` — only handles POST:
    1. Parse body, validate refresh_token is present → validationErrorResponse if missing
    2. Hash the received token: $receivedHash = hash('sha256', $body['refresh_token'])
    3. Look up in DB: $row = $this->refreshTokenRepo->findByHash($receivedHash)
    4. If $row is null → return errorResponse 401 'INVALID_TOKEN' 'Invalid refresh token'

    7. Revoke old refresh token (rotation): $revoked = $this->refreshTokenRepo->revoke((int) $row->id)
       - If $revoked is false (already revoked — replay attack) → return errorResponse 401 'TOKEN_REVOKED' 'Refresh token has been revoked'
    8. Look up user: $user = $this->userRepository->findById((int) $row->user_id) — need UserRepositoryInterface in constructor too
       - If user is null → return errorResponse 500 'INTERNAL_ERROR' 'User not found'
    9. Generate new access token: $accessToken = $this->tokenService->generateToken(['user_id' => $user->id, 'level_id' => $user->levelId, 'username' => $user->username], $this->settings['jwt']['access_token_expiry'] ?? 900)
    10. Generate new refresh token: $refreshData = $this->tokenService->generateRefreshToken()
    11. Store new refresh token hash: $expiresAt = new \DateTimeImmutable('+' . ($this->settings['jwt']['refresh_token_expiry'] ?? 604800) . ' seconds'); $this->refreshTokenRepo->store($user->id, $refreshData['hash'], $expiresAt)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Information Disclosure | Refresh token in transit | mitigate | Refresh tokens sent over HTTPS only (production). In development, tokens visible in response body — acceptable for test project. |
| T-03-16 | Spoofing | RefreshTokenAction token validation | mitigate | Store SHA-256 hashes of refresh tokens (not raw tokens). Compare received hash against stored hash. Raw token is never stored in DB. |
| T-03-17 | Tampering | Refresh token replay attack | mitigate | Single-use refresh tokens: revoke old token BEFORE issuing new pair. Check affected rows on revoke — if 0, token was already revoked (replay detected), return 401. |
| T-03-18 | Information Disclosure | LoginAction response | mitigate | Response includes refresh_token in body — ensure HTTPS in production. Token is 64-char random string, not guessable. |
| T-03-19 | Tampering | TokenService generateRefreshToken | mitigate | Uses random_bytes(32) — cryptographically secure random source. SHA-256 hash for storage. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed; uses existing firebase/php-jwt for JWT encoding |
</threat_model>
